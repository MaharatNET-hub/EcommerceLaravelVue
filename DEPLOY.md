# النشر على Render

هذا المشروع جاهز للنشر على [Render](https://render.com) عبر Blueprint. يوجد إعدادان جاهزان حسب حاجتك:

## 1) `render.yaml` — تجربة مجانية (بدون بطاقة دفع)

خدمة ويب واحدة فقط (Docker) على الخطة المجانية، تستخدم **SQLite** بدلًا من MySQL (لأن الخدمات الخاصة/Private Services على Render غير متاحة إلا على الخطط المدفوعة، فما قدرنا نستضيف MySQL ذاتيًا مجانًا).

**تحذير مهم**: لا يوجد قرص دائم، فكل البيانات (منتجات، طلبات، صور مرفوعة) تُمسح تلقائيًا:
- عند كل عملية نشر جديدة (Deploy)
- وعند كل مرة تدخل فيها الخدمة بوضع الخمول (Inactivity) بعد فترة من عدم الاستخدام ثم تصحى (Render يشغّل حاوية جديدة من الصورة الأصلية).

بيانات المتجر التجريبية تُعاد زراعتها تلقائيًا في كل مرة (فالموقع ما بيضل فاضي)، لكن أي طلب أو تعديل حقيقي تعمله بين فترة وأخرى رح يضيع. **مناسب للتجربة فقط، غير مناسب لمتجر حقيقي فعّال.**

## 2) `render.paid.yaml` — إعداد جاهز للاستخدام الفعلي (يتطلب بطاقة دفع)

خدمتان: الموقع + قاعدة MySQL مستضافة ذاتيًا، كلاهما بقرص دائم فلا تضيع أي بيانات أو صور. يتطلب خطة Starter المدفوعة (فوترة بالثانية، ~7$/شهر لكل خدمة تقريبًا).

للتفعيل: أعد تسمية الملف إلى `render.yaml` (أو حدد مساره كـ Blueprint Path عند الإعداد على Render)، ثم انشر الـ Blueprint.

---

## خطوات النشر (تنطبق على الإعدادين)

1. ادخل إلى [dashboard.render.com](https://dashboard.render.com) وسجّل الدخول.
2. اختر **New > Blueprint**.
3. اربط حساب GitHub وحدد هذا المستودع (`MaharatNET-hub/EcommerceLaravelVue`) والفرع الذي يحتوي هذا الكود.
4. سيكتشف Render ملف `render.yaml` تلقائيًا ويعرض الخدمة/الخدمتين — اضغط **Apply**.
5. بعد اكتمال أول Deploy:
   - افتح خدمة `shop-app` في Render، وفي تبويب **Environment** اضبط `APP_URL` على الرابط الذي أعطاك إياه Render (مثل `https://shop-app-xxxx.onrender.com`)، ثم أعد النشر (Manual Deploy) ليُطبّق الرابط في الروابط المولّدة و SEO.
   - في نفس التبويب ستجد `ADMIN_EMAIL` و `ADMIN_PASSWORD` (تم توليدها تلقائيًا) — استخدمها لتسجيل الدخول إلى لوحة التحكم على `/admin`.
6. إن أردت ربط نطاق مخصص، أضِفه من تبويب **Settings > Custom Domains** في خدمة `shop-app`.

## ملاحظات مهمة

- الدفع عند الاستلام (COD) هو طريقة الدفع الوحيدة المفعّلة حاليًا؛ لإضافة بوابة دفع إلكتروني (Stripe/PayPal/HyperPay...) يلزم إضافة مفاتيح API الخاصة بها كمتغيرات بيئة وربطها بمنطق `CheckoutController`.
- البريد الإلكتروني (تأكيد الطلبات، إعادة تعيين كلمة المرور) يستخدم حاليًا `MAIL_MAILER=log`؛ لتفعيل إرسال بريد حقيقي أضف بيانات SMTP كمتغيرات بيئة في خدمة `shop-app`.
- لتشغيل المشروع محليًا للتطوير: `composer install && npm install && php artisan migrate --seed && npm run dev` (مع `php artisan serve` في نافذة أخرى).
